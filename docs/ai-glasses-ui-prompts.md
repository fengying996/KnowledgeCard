# AIUI 眼镜学习卡片 UI 风格提示词沉淀

本文根据一组眼镜 UI 截图抽象出可复用的视觉语言、交互模式和提示词模板，并将主题重构为“学习卡片”，便于后续让 AI 生成更贴近 AI 眼镜场景的学习类页面。

## 1. 风格定位

这组界面的核心气质不是传统手机 App，也不是复杂 3D 大屏，而是：

- 面向 AR / AI 眼镜的轻量信息 HUD
- 高透叠加式界面，不遮挡真实环境
- 单色高识别视觉系统
- 强状态感、弱装饰感
- 适合远看、扫读、手势快速切页

一句话概括：

> 半透明现实世界背景上的荧光绿单色学习信息 HUD，带像素感数字字体、细描边卡片、低负担信息分层和适合眼镜交互的滑动切页结构。

## 2. 视觉语言拆解

## 2.1 配色

- 主色为高亮荧光绿，接近 `#39FF5A`、`#32FF4E`、`#42F56C`
- 背景不是纯黑，而是“真实世界画面 + 极轻 UI 叠加”
- 尽量避免大面积实心底色，优先使用：
  - 细线描边
  - 半透明蒙层
  - 低透明度块面
  - 点状或虚线分隔
- 颜色体系尽量克制，核心建议：
  - 主信息：荧光绿
  - 次信息：低亮度绿
  - 弱化信息：半透明绿
  - 警示信息如必须出现，也尽量先用亮度对比而不是多彩色

## 2.2 字体与数字风格

- 标题和编号采用像素感 / 数码管感 / 电子记分牌感字体
- 中文正文仍可读，但建议略带工业感和细窄结构
- 数字需要特别醒目，适合远距识别
- 文案层级建议：
  - 顶部标题：中等偏大，字距略开
  - 核心标题或编号：最大
  - 时间、标签、状态：中小号
  - 辅助说明：更轻、更细

关键词：

- pixel scoreboard font
- retro digital sports display
- narrow geometric Chinese UI text
- monospace numeric emphasis

## 2.3 结构与边框

- 版式以横向卡片为主，符合眼镜视野展开习惯
- 卡片使用细描边矩形，圆角非常轻
- 分割线偏爱点线、虚线、弱实线
- 模块之间留白明显，避免拥挤
- 内容整体以“中心聚焦 + 分区对照”为主，尤其适合概念、定义、要点、对比、记忆强化

典型结构：

- 顶部：学习主题 / 页面标题
- 中部：核心知识点、标题、状态
- 下部：例句、步骤、总结、交互提示
- 底部：手势提示、语音唤醒提示、返回提示

## 2.4 图标与图形

- 图标风格尽量单线条、像素边缘、低装饰
- 图标建议做单色化、线稿化、矩形框内展示
- 图表和辅助图形不追求复杂渐变，优先：
  - 单线流程图
  - 细线分割图
  - 点状分页指示器
  - 双栏对照结构

避免：

- 厚重拟物图标
- 多彩渐变图表
- 高饱和背景图块
- 玻璃拟态大面积模糊

## 3. 交互模式拆解

从截图能抽象出很适合眼镜 UI 的几个交互原则。

## 3.1 一屏一任务

- 每一屏只强调一个核心任务
- 例如：
  - 学习卡片列表
  - 知识点详情
  - 重点对比
  - AI 总结
  - AI 讲解
- 单页信息量受控，支持快速扫读，不追求一次展示全部内容

## 3.2 横向切页 / 分屏浏览

- 通过滑动切换不同信息页，而不是在单页堆叠所有模块
- 非常适合眼镜端，因为：
  - 用户停留时间短
  - 视野受限
  - 手势操作适合左右翻页

可抽象成：

- Page 1: 学习卡片列表
- Page 2: 知识点详情
- Page 3: 重点对比
- Page 4: AI 总结
- Page 5: AI 讲解

## 3.3 持续存在的操作提示

- 底部始终保留极简操作提示
- 提示文案短而固定，例如：
  - 滑动切换页面
  - 单击 / 滑动切换页面
  - 双击触控板返回上一级
  - 呼叫语音助手查看更多

这类提示非常符合眼镜设备，不需要用户记忆复杂交互。

## 3.4 中心聚焦

- 重要内容总放在视觉中心附近
- 核心概念、关键词、结构图、讲解正文都以中心区域为锚点
- 用户无需大幅转动视线

## 3.5 低干扰动画

- 适合眼镜端的动画不是炫酷，而是轻量反馈
- 建议只保留：
  - 页面切换淡入淡出
  - 指示点变化
  - 数字轻微跳变
  - 图表线条渐显
- 避免大面积缩放、旋转、持续闪烁

## 4. 适合眼镜 UI 的设计原则

这组风格之所以适合眼镜，是因为它天然满足下面几点：

- 高对比：荧光绿在复杂真实背景上依然容易识别
- 低遮挡：细边框和透明背景不会遮住现实视野
- 易扫读：信息模块化明显，用户一眼知道先看哪里
- 强结构感：标题、关键词、步骤、总结天然适合 HUD 呈现
- 易语音协同：底部可持续放置语音唤醒提示

## 5. 风格提示词

下面的提示词适合用于让 AI 生成：

- AIUI 页面设计描述
- UI 截图风格方案
- 前端页面结构草案
- `.ink` 页面实现提示

## 5.1 总风格提示词

```text
请生成一个适用于 AI 眼镜 / AR HUD 的学习卡片界面，整体采用高透明叠加式设计，背景默认视为真实世界画面。UI 使用单色荧光绿视觉系统，细线描边卡片、像素感数字与标题样式、弱实底、点状分隔线、中心聚焦排版。页面信息应适合远距离扫读，减少装饰，突出知识标题、关键词、步骤、总结和手势提示。整体风格像未来感学习 HUD，而不是手机 App 或传统网页后台。
```

## 5.2 强化版风格提示词

```text
设计一个适合智能眼镜显示的学习卡片界面。要求：1）荧光绿色单色 HUD 风格；2）透明背景叠加真实环境；3）所有模块以细描边矩形和轻圆角组织；4）标题和编号使用像素记分牌风格字体；5）信息结构简洁、中心对齐、分区清晰；6）图标做单色线稿化处理；7）底部保留手势操作提示和语音唤醒提示；8）页面支持左右滑动切换不同学习页；9）整体科技感强，但克制、轻薄、不花哨。
```

## 5.3 页面生成提示词模板

```text
请基于“荧光绿单色 AR 眼镜 HUD”风格生成页面，要求：
- 真实世界透视背景上叠加 UI
- 主色为 neon green
- 模块边框细、信息分层清晰
- 适合眼镜端远视阅读
- 页面底部始终包含操作提示
- 动效轻量，只做淡入、滑动切页和数字更新
- 不使用多彩渐变、不使用厚重阴影、不使用手机卡片式大色块
```

## 6. 分场景提示词模板

## 6.1 学习卡片列表页

```text
请生成一个 AI 眼镜端的学习卡片列表页。界面为荧光绿单色 HUD 风格，顶部显示学习主题和当前日期，中间以横向细描边卡片展示多张学习卡片，每张卡片包含知识点标题、学习进度、难度、预计时长和单色线稿图标。卡片层次清晰，支持快速扫读，适合眼镜小视野场景。底部保留“滑动切换页面”和“返回上一级”的提示。
```

## 6.2 知识点详情页

```text
请生成一个智能眼镜 HUD 风格的知识点详情页。顶部显示课程或主题名称，中间突出核心概念标题、当前学习状态、关键词和简短定义；下方展示学习目标、关键公式或关键步骤等信息。整体采用透明叠加、荧光绿单色、像素编号字体、中心聚焦布局，并保留分页指示器和底部手势提示。
```

## 6.3 重点对比页

```text
请生成一个 AI 眼镜风格的重点对比页。页面中轴垂直列出对比维度，如定义、特征、适用场景、常见误区、记忆口诀；左右两侧分别显示两个相近知识点或概念，形成镜像对照布局。使用单色荧光绿细字和点状分隔线，弱化背景、强化知识差异，保证远距离可读。
```

## 6.4 AI 总结页

```text
请生成一个智能眼镜端的学习总结页。顶部展示当前主题和学习完成度，中部展示知识要点数量、掌握程度、易错点提醒以及简洁单线图表，支持“今日复习建议”“记忆强度”“推荐下一张卡片”等模块。风格必须是透明 AR HUD、荧光绿单色、细描边卡片、轻量科技感、低干扰信息密度。
```

## 6.5 AI 讲解页

```text
请生成一个 AI 眼镜端的学习讲解页。页面以大段可读文本为主，但必须控制阅读疲劳：使用较大行高、均匀左右留白、轻边框正文容器、清晰标题和底部操作提示。整体仍保持荧光绿单色 HUD 风格，像知识讲解投影层，适合用户在现实场景中边看边听语音讲解。
```

## 7. 可直接给 AIUI 代码生成器的提示词

```text
请用 AIUI 生成一个适用于智能眼镜的学习卡片多页界面，包含“学习卡片列表页、知识点详情页、重点对比页、AI 总结页、AI 讲解页”。视觉风格要求：
- 透明叠加式 HUD
- 荧光绿单色主题
- 细描边卡片和点状分隔线
- 像素编号风格数字
- 中心聚焦、分区清晰、适合远视阅读
- 底部固定显示手势提示与语音提示

交互要求：
- 支持左右滑动分页
- 支持分页指示点
- 支持双击返回上一级
- 每页只承载一个核心学习任务

实现要求：
- 优先使用简单布局，不堆砌视觉特效
- 尽量通过边框、留白、字号层级体现信息分组
- 动画只使用轻量切页和淡入淡出
- 所有文案和图标都服务于“快速扫读”
```

## 8. 反向约束提示词

为了避免生成结果跑偏，建议在提示词里加入这些限制。

```text
不要做成手机 App 风格
不要使用多彩渐变和高饱和配色
不要使用厚重投影和拟物图标
不要让背景面板过厚过实
不要堆叠过多模块在同一屏
不要使用复杂动画和大面积旋转缩放
不要做成传统后台数据看板
```

## 9. 关键词速查

适合在提示词中混合使用的关键词：

- neon green monochrome overlay
- transparent real-world background
- pixel scoreboard typography
- thin outline cards
- lightweight futuristic learning UI

- structured knowledge comparison

## 10. 推荐的最终组合提示词

```text
请设计一个适用于 AI 眼镜的学习卡片界面，采用 transparent AR HUD 风格，叠加在真实环境背景之上。整体使用 neon green monochrome 视觉系统，thin outline cards、pixel scoreboard typography、dotted separators、center-focused composition、structured knowledge comparison。页面需要适合远视扫读和手势交互，支持左右滑动切换“卡片列表、知识点详情、重点对比、AI总结、AI讲解”五类页面。避免手机 App 化、避免多彩渐变、避免厚重面板，强调轻量、清晰、未来感和低遮挡。
```

## 11. 结论

这组截图最值得保留的不是某个具体模块，而是一整套适合眼镜设备的学习信息表达方式：

- 单色高对比
- 透明低遮挡
- 中心聚焦
- 分页浏览
- 强结构信息
- 极简交互提示

后续如果要扩展到别的学习题材，比如单词记忆、公式卡片、历史时间线、编程概念、考试复盘，也建议沿用这套抽象，而不是回到手机 UI 的设计惯性。
